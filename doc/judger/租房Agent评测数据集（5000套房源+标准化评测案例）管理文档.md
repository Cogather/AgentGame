# 二、租房Agent评测标准化打分体系

本体系采用「1-10分制」，打分维度与Agent评测点强绑定，根据评测用例类型（简单/复杂）差异化侧重，分数定义统一，可实现自动化量化评估，确保评测结果公平、可对比。

|分数区间|打分定义|适用场景说明|
|---|---|---|
|9-10分|完全匹配/最优策略|简单类：所有用户明确条件100%满足，无遗漏/错误；复杂类：模糊语义量化合理、条件权衡优先级符合常识、隐性需求挖掘准确、交互澄清到位|
|7-8分|核心匹配/良好策略|简单类：核心条件（位置/户型/预算）匹配，1个次要条件（楼层/朝向/装修）轻微不符；复杂类：模糊语义量化基本合理、仅1个非核心维度策略偏差，不影响核心推荐|
|5-6分|部分匹配/一般策略|简单类：核心条件1个不符，或2-3个次要条件不符；复杂类：模糊语义量化有偏差、条件权衡优先级轻微不合理、隐性需求挖掘不完整|
|1-4分|核心不匹配/错误策略|简单类：2个及以上核心条件不符，或排除条件未过滤；复杂类：模糊语义量化错误、条件权衡优先级严重不合理、未挖掘隐性需求/未澄清模糊指令、推荐失效房源|
## 打分核心维度

打分过程围绕以下5个核心维度展开，全面评估Agent的核心能力，确保评测全面、精准：

1. 条件解析准确性：是否正确理解用户的显式/隐式需求，无解析偏差；

2. 结果过滤精准性：是否按解析后的条件过滤房源，无无关/失效房源，无遗漏符合条件的房源；

3. 策略合理性：多条件冲突/模糊需求时，妥协/量化/挖掘的策略是否符合租房常识，贴合用户实际需求；

4. 交互完整性：信息不足/条件矛盾时，是否主动澄清关键信息，无无效回复、无冗余询问；

5. 解释清晰性：推荐房源时，是否明确说明匹配点/妥协点，理由是否合理、易懂，便于用户理解推荐逻辑。
